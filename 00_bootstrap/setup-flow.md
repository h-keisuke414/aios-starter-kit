# 初回セットアップ質問フロー

Claude はこのファイルの順に質問を投げ、回答を [../client-overview.md](../client-overview.md) と各テンプレに反映する。
**1 問ずつ、または 3 問ずつ、答えやすい単位で区切って聞く**。「分からない・後で」は許容。

---

## ブロック A: 案件種別の確認（1 問）

1. **新規案件ですか？既存案件の続きですか？**
   - 新規 → ブロック B から開始
   - 既存 → スキップして [../02_roadmap/](../02_roadmap/) の当月ファイルを開く

---

## ブロック B: クライアント基本情報（7 問）

| # | 質問 | 反映先 |
|---|---|---|
| 2 | 会社の正式名称（株式会社○○ 等） | `[CLIENT_NAME]` 全置換 |
| 3 | 業種（製造 / 飲食 / 小売 / 建設 / 美容 / 士業 / 介護 / その他） | `[INDUSTRY]` 全置換 |
| 4 | 所在地（都道府県・市区町村） | `client-overview.md` |
| 5 | 主担当者の氏名・役職 | `[OWNER]` 全置換 + `client-overview.md` |
| 6 | 契約期間（例: 6 ヶ月） | `[CONTRACT_PERIOD]` 全置換 |
| 7 | 開始日（YYYY-MM-DD） | `[KICKOFF_DATE]` 全置換 |
| 8 | 月額契約金額（任意） | `[MONTHLY_RATE]` 全置換 |

**回答後の自動処理**:
```bash
LC_ALL=C find . -type f -name "*.md" -not -path "./.git/*" -not -path "./00_bootstrap/*" \
  -exec sed -i '' "s/\[CLIENT_NAME\]/${回答}/g" {} +
```
（プレースホルダごとに同様に実行。Claude は 1 回ごとに置換結果を確認）

---

## ブロック C: 会社規模・体制（4 問）

| # | 質問 | 用途 |
|---|---|---|
| 9 | 従業員数（おおよそで OK） | フェーズ計画の規模感 |
| 10 | IT に詳しい人は社内にいるか（はい / いいえ / 1 人だけいる） | 研修密度の調整 |
| 11 | 現在使っている主なツール（Excel / LINE / メール / 会計ソフト名 等） | `00_discovery/current-state-assessment.md` |
| 12 | AI 導入で **一番解決したい困りごと**（1 つで OK） | `01_strategy/use-case-candidates.md` |

---

## ブロック D: プラグイン・ローカル資料（2 問）

| # | 質問 | 次のアクション |
|---|---|---|
| 13 | Claude Code のプラグインインストール状況を確認しても良いか | [plugins-install.md](plugins-install.md) を実行 |
| 14 | クライアント PC 上に過去のヒアリング議事録・社内マニュアル等の参考資料があるか | [local-resources.md](local-resources.md) を実行 |

---

## ブロック E: 完了処理

すべて終わったら Claude は以下を実行:

1. `.local/` ディレクトリを作成（存在しなければ）
2. `.local/.bootstrapped` ファイルに今日の日付を書き込む
3. ユーザーに完了報告:
   > 「初回セットアップが完了しました。次は地域・業界の事前リサーチを行いますか？（[00_discovery/local-research.md](00_discovery/local-research.md)）」

---

## 質問例（実際の会話文体）

> Claude: 「初回セットアップを始めます。まず会社の正式名称を教えてください。」
> ユーザー: 「株式会社森田製作所です」
> Claude: 「ありがとうございます。プレースホルダを `株式会社森田製作所` に一括置換しました。次に業種を教えてください。製造業でしょうか？」

このように **1 問ずつ確認 → 置換 → 次へ** のリズムで進める。
