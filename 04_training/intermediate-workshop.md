# 中級者向けワークショップ（120分）

> M3 で実施。基本操作は習得済みの参加者を対象に、業務応用・複数ステップ自動化を扱う。

## ゴール

- 自分の業務フローを「AI に任せる部分」「人間が判断する部分」に分解できる
- 複数ステップのプロンプトを設計できる
- 「うまくいかない」原因を切り分けて改善できる

## タイムテーブル

| 時間 | 内容 |
|---|---|
| 0:00 | M2 振り返り・成果共有（15分） |
| 0:15 | 業務フロー分解ワーク（30分） |
| 0:45 | 複数ステップのプロンプト設計（30分） |
| 1:15 | 失敗ケース改善ワーク（20分） |
| 1:35 | データ活用入門（15分） |
| 1:50 | クロージング・宿題（10分） |

## ワーク1: 業務フロー分解（30分）

各自の業務を1つ選び、以下のフォーマットで書き出す:

| ステップ | 内容 | AI 任せる | 人間判断 | 理由 |
|---|---|---|---|---|
| 1 | | ○ / × | ○ / × | |
| 2 | | | | |

3人ペアで共有・フィードバック。

## ワーク2: 複数ステップ・プロンプト設計（30分）

「議事録 → 要約 → アクションアイテム抽出 → 担当者宛メール下書き」のような連鎖を1つ作る。

[../03_knowledge/prompt-templates/meeting-summary.md](../03_knowledge/prompt-templates/meeting-summary.md) を参考に。

## ワーク3: 失敗ケース改善（20分）

「うまくいかなかったプロンプト」を持ち寄り、改善案をその場で考える。

改善視点:
1. 役割が不明確 → 「あなたは○○です」を追加
2. 出力形式が曖昧 → 表 / 箇条書きを指定
3. インプット情報が不足 → 補足情報を追加
4. ステップが多すぎる → 分割実行

## ワーク4: データ活用入門（15分）

[../03_knowledge/prompt-templates/data-analysis.md](../03_knowledge/prompt-templates/data-analysis.md) のパターンを紹介。

「自分の手元データで試したいもの」をメモ。

## 宿題

- 30日チャレンジの後半15日のテーマ
- 新しく作ったプロンプトを1つ [../03_knowledge/prompt-library.md](../03_knowledge/prompt-library.md) に登録
