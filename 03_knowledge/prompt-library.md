# 案件固有プロンプト・ライブラリ

> 運用中に「これは便利」と判断されたプロンプトを蓄積するファイル。
> 案件特有のドメイン用語・フォーマットを含む。
> このファイルが充実すると、**M6 終了後にクライアントが内製化できる資産** になる。

---

## 使い方

- 新しいプロンプトを追加する際は下記テンプレートで書く
- 動かなくなった or 改善されたものは削除せず「v2」として追加し、旧版は折りたたむ
- **月次レポート時に必ず棚卸し**: 使われていないプロンプトは整理、よく使うプロンプトは「定番」マークを付ける

---

## プロンプトを追加すべきタイミング

| シーン | 例 |
|---|---|
| 現場が「便利！」と言った時 | 「議事録のまとめ方、毎回これでお願い」 |
| 同じ作業を 3 回以上繰り返した時 | 求人原稿・商品説明文・FAQ など |
| クライアント独自フォーマットがある時 | 「営業日報はこの形式で出す」 |
| ベテランの判断パターンを再現したい時 | 「○○さんならこういう返信をする」 |

---

## 設計のコツ

### 1. 役割（Role）と前提条件を最初に書く

❌ 悪い例:
```
議事録作って
```

✅ 良い例:
```
あなたは中小製造業の業務改善を支援するアシスタントです。
以下の音声文字起こしから、株式会社○○の経営会議議事録を作成してください。
出力フォーマット: 決定事項 / ToDo（担当・期限）/ 論点 / 次回アクション
```

### 2. 出力フォーマットを明示する

「議事録っぽく」ではなく「以下の見出しで」と書く。

### 3. NG パターンを書く

「専門用語は避ける」「会社名は伏せる」など、避けて欲しい挙動を先に伝える。

### 4. 例示する（Few-shot）

```
例 1:
入力: [サンプル入力]
出力: [サンプル出力]
```

短いサンプルでも精度が大きく上がる。

## テンプレート

````markdown
### [P-XXX] プロンプト名

**用途**: 何のために使うか（1 行）
**使用ツール**: Claude / ChatGPT / Gemini / NotebookLM
**頻度**: 毎日 / 毎週 / 毎月 / 都度
**作成日**: YYYY-MM-DD
**作成者**:
**バージョン**: v1
**定番マーク**: ☆（よく使うものに付ける）

#### プロンプト本文

```
（プロンプトをここに貼る。改行・空行はそのまま）
```

#### 入力例

```
（実際の入力例 1〜2 個）
```

#### 期待される出力

```
（出力サンプル）
```

#### 注意点

- 機密データを入れる場合の匿名化方法
- 失敗ケース（こうすると変な出力になる）
- バージョン履歴
````

## 業種別の出発点（参考プロンプト集）

案件で実際に使ったものではないが、ここからアレンジすると早い。

### 製造業

- 不良品報告書のドラフト生成
- 受注メールから出荷指示書への変換
- 安全教育マニュアルの平易化（外国人スタッフ向け）

### 飲食・小売

- 新メニュー説明文（POP・SNS・公式サイト用に同時生成）
- お客様クレーム対応の返信下書き
- 仕入れ価格交渉メールの草案

### 建設

- 見積書の項目チェック・抜け漏れ確認
- 安全パトロール報告書の構造化
- 協力会社への発注書ドラフト

### 美容・サロン

- 顧客カルテからの次回提案メッセージ
- SNS 投稿の月次ネタ出し
- 来店御礼メッセージ（リピート促進）

### 士業（税理士・社労士・行政書士）

- 顧問先からの質問への一次回答ドラフト
- 法改正情報のニュースレター
- 申請書類のチェックリスト

詳細プロンプト本体は [prompt-templates/](prompt-templates/) を参照。
