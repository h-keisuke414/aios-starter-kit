# キックオフチェックリスト

> 「後から変更すると痛い」ものを最初に決め切るためのチェックリスト。
> M0 のうちに以下すべてに○がつくこと。

## 1. スコープ

- [ ] 対象業務範囲（どの部署・どの業務か）
- [ ] 対象外（やらないこと）の明示
- [ ] 関与する人の範囲（推進者 / 現場 / 決裁者）

## 2. AI ツール選定

- [ ] 使用許可される AI（Claude / ChatGPT / Gemini / Copilot 等）
- [ ] 法人契約 or 個人契約 or BYO どれにするか
- [ ] 機密データを入力可能な環境か（Enterprise契約 / API / データ学習無効）

## 3. 機密情報の境界

- [ ] AI に入れて良い情報・ダメな情報の線引き
- [ ] 顧客個人情報の扱い
- [ ] 取引先情報・契約金額の扱い
- [ ] 入力前の匿名化ルール

## 4. 成果指標（KPI）

- [ ] 何で「うまくいった」と判断するか（時間削減 / 件数 / 品質）
- [ ] 計測方法（タイムトラッキング / アンケート等）
- [ ] ベースライン（現状の所要時間・件数）測定タイミング

## 5. 推進体制

- [ ] 推進者（クライアント側）の確定
- [ ] 現場リーダー（実際に使う人）の確定
- [ ] 定例ミーティングの曜日・時間
- [ ] エスカレーションパス

## 6. 環境・予算

- [ ] AI ツール利用料の負担者・支払いフロー
- [ ] 必要な PC / 環境（社内ネットワーク制約）
- [ ] 既存システム連携の可否（社内SE 同席要否）

## 7. 期待値と進め方

- [ ] 6ヶ月後の理想状態を1文で言語化
- [ ] M3 までに止めても OK な失敗ライン
- [ ] 内製化のゴール設定（属人化させない）
